# The system of accounting and management of personnel data of freelancers

## О проекте

Целью данной курсовой работы являлось проектирование и создание базы данных, а также разработка desktop-приложения на Windows Forms, обеспечивающего эффективное управление информацией о фрилансерах, нанимателях, проектах и платежах.

В рамках работы был проведён анализ требований пользователей, разработана структура базы данных, реализован функционал для пользователей (нанимателей и фрилансеров) и обеспечена возможность взаимодействия между ними. Особое внимание было уделено созданию интуитивного и удобного интерфейса.

## Технологический стек

*   **Backend:** C# (.NET Framework)
*   **Пользовательский интерфейс:** Windows Forms
*   **База данных:** Microsoft SQL Server
*   **Протокол работы с БД:** ADO.NET (SqlConnection, SqlCommand, SqlDataReader)

### Для неавторизованных пользователей
*   Регистрация в системе (как Фрилансер или как Наниматель)
*   Вход в систему (аутентификация)

### Для Фрилансеров
*   **Управление профилем:** Редактирование личной информации, навыков, опыта работы, часовой ставки и биографии.
*   **Управление учётной записью:** Изменение логина, email, номера телефона.
*   **Поиск проектов:** Просмотр всех доступных проектов.
*   **Управление предложениями:** Подача заявок на проекты (создание предложений), просмотр статуса своих предложений.
*   **Управление контрактами:** Просмотр активных и завершённых контрактов.
*   **Просмотр платежей:** История полученных платежей по контрактам.

### Для Нанимателей
*   **Управление профилем компании:** Редактирование названия и информации о компании.
*   **Управление учётной записью:** Изменение логина, email, номера телефона.
*   **Управление проектами:** Создание, просмотр и обновление проектов (изменение статуса, завершение).
*   **Управление предложениями:** Просмотр предложений на свои проекты, принятие или отклонение заявок от фрилансеров.
*   **Управление контрактами:** Создание контрактов на основе принятых предложений, просмотр и управление контрактами.
*   **Управление платежами:** Создание и отслеживание платежей по контрактам.

## Структура базы данных

![image](https://github.com/user-attachments/assets/a32f14ee-92c4-45fd-a1ce-fb62987ab461)

База данных состоит из следующих таблиц (сущностей):

### 1. Пользователи (Users)
*   `Id пользователя` (INT, PK, Identity) - Первичный ключ.
*   `Имя пользователя` (NVARCHAR(20)) - Логин пользователя.
*   `Пароль` (BINARY(32)) - Хэш пароля (алгоритм SHA-256).
*   `Роль` (NVARCHAR(20)) - Роль пользователя: 'Фрилансер' или 'Наниматель'.
*   `Электронная почта` (NVARCHAR(30))
*   `Номер телефона` (NVARCHAR(20))

### 2. Фрилансеры (Freelancers)
*   `Id фрилансера` (INT, PK, Identity) - Первичный ключ.
*   `Id пользователя` (INT, FK) - Внешний ключ на `Пользователи`.
*   `Навыки` (NVARCHAR(1000)) - Список навыков фрилансера.
*   `Опыт работы` (NVARCHAR(10)) - Опыт работы (например, "5 лет").
*   `Часовая ставка` (INT) - Почасовая ставка оплаты.
*   `Информация о себе` (NVARCHAR(1000)) - Биография/описание.

### 3. Наниматели (Employers)
*   `Id нанимателя` (INT, PK, Identity) - Первичный ключ.
*   `Id пользователя` (INT, FK) - Внешний ключ на `Пользователи`.
*   `Название компании` (NVARCHAR(100))
*   `Информация о компании` (NVARCHAR(1000))

### 4. Проекты (Projects)
*   `Id проекта` (INT, PK, Identity) - Первичный ключ.
*   `Id нанимателя` (INT, FK) - Внешний ключ на `Наниматели`.
*   `Название проекта` (NVARCHAR(100))
*   `Описание проекта` (NVARCHAR(1000))
*   `Бюджет` (INT)
*   `Срок выполнения` (DATETIME)
*   `Статус проекта` (NVARCHAR(20)) - Например, "Открыт", "В работе", "Завершён", "Отменён".

### 5. Предложения (Bids)
*   `Id предложения` (INT, PK, Identity) - Первичный ключ.
*   `Id проекта` (INT, FK) - Внешний ключ на `Проекты`.
*   `Id фрилансера` (INT, FK) - Внешний ключ на `Фрилансеры`.
*   `Текст предложения` (NVARCHAR(1000)) - Сопроводительное письмо фрилансера.
*   `Сумма ставки` (INT) - Предлагаемая фрилансером сумма за проект.
*   `Статус предложения` (NVARCHAR(20)) - "На рассмотрении", "Принято", "Отклонено".

### 6. Контракты (Contracts)
*   `Id контракта` (INT, PK, Identity) - Первичный ключ.
*   `Id проекта` (INT, FK) - Внешний ключ на `Проекты`.
*   `Id фрилансера` (INT, FK) - Внешний ключ на `Фрилансеры`.
*   `Дата начала` (DATETIME)
*   `Дата окончания` (DATETIME)
*   `Статус контракта` (NVARCHAR(20)) - "Активен", "Завершён", "Расторгнут".

### 7. Платежи (Payments)
*   `Id платежа` (INT, PK, Identity) - Первичный ключ.
*   `Id контракта` (INT, FK) - Внешний ключ на `Контракты`.
*   `Сумма платежа` (INT)
*   `Дата платежа` (DATETIME)
*   `Статус платежа` (NVARCHAR(20)) - "Ожидает", "Выполнен", "Отменён".


## Безопасность

*   Пароли пользователей не хранятся в открытом виде. Они хэшируются с использованием алгоритма **SHA-256** перед сохранением в базу данных (`Init.sha256_hash`).
*   Для предотвращения SQL-инъекций все пользовательские данные передаются в запросы через параметризованные запросы (`SqlParameter`).

## Запуск проекта

1.  **Требования:** Установленные .NET Framework и SQL Server.
2.  **Восстановление БД:** Необходимо создать базу данных и выполнить скрипты для создания таблиц, связей, хранимых процедур и триггеров (скрипты должны прилагаться отдельно).
3.  **Настройка подключения:** В коде должен быть класс `ConnectionManager`, отвечающий за подключение к SQL Server. Необходимо убедиться, что строка подключения в нём верная.
4.  **Сборка:** Открыть решение (.sln) в Visual Studio и скомпилировать проект.
5.  **Запуск:** Запустить скомпилированное приложение.
